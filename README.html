LUFS Lens

Loudness sanity check for streaming releases

You spent hours perfecting your master.
Spotify will spend 0.3 seconds deciding it was too loud.

LUFS Lens lets you see the verdict before the algorithm does.

What it does

LUFS Lens analyzes your audio files and checks:

  • Integrated LUFS (overall loudness)
  • True Peak (inter-sample clipping risk)
  • Sample Peak
  • LRA (Loudness Range)
  • Sample rate and technical metadata
Typical streaming reference:
Around -14 LUFS integrated with true peak at or below -1 dBTP.

Why -14 LUFS?

Most streaming platforms normalize playback loudness. If your track is louder than their target, it will simply be turned down. Over-limiting therefore does not make your track louder — it only reduces dynamics and increases distortion risk.

Checking before upload helps preserve punch and clarity.

How to use

Option A – Double-click

  • Open LUFS Lens.bat
  • Select one or multiple files
  • The report opens automatically

Option B – Drag & Drop

  • Drag a file or folder onto LUFS Lens.bat
Your Files → LUFS Lens → Analysis → HTML Report → Streaming Platform

Where are reports saved?

All reports are saved inside the tool folder:

LUFSLens_v1\Reports\

  • loudness_report_TIMESTAMP.html
  • loudness_report_TIMESTAMP.csv

HTML = visual overview.
CSV = spreadsheet-friendly data.

Understanding the Metrics

Integrated LUFS
Overall average loudness of the track. If significantly above -14 LUFS, it will likely be turned down.

True Peak (dBTP)
Measures inter-sample peaks that can exceed digital 0 dB. Keeping it at or below -1 dBTP helps prevent distortion after encoding.

Sample Peak (dBFS)
Highest raw digital sample value. Useful for detecting hard clipping.

LRA (Loudness Range)
Indicates dynamic variation. Low LRA = heavily compressed. Higher LRA = more dynamic contrast.

Status Verdict

READY
Within LUFS tolerance, safe true peak, correct sample rate.
ADJUST
Outside recommended values. Check the Issues column in the report.
ERROR
Analysis failed. Check file integrity or format.
